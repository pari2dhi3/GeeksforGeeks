                                                                                                                                      Garima Dhanania akamal8 5 months ago

                                                                                                                                      The problem arises not when it is leaf whose value exactly equals the sum.

                                                                                                                                      For that, that would be taken care of by the base case if(root==NULL). When you call hasPathSum on root->left and root->right on a node which has no left or right child, it is essentially called on NULL nodes, so both calls return 1 if sum-curr==0 i.e if sum==curr node-> data which is the case you are talking about.

                                                                                                                                      The above code creates a problem when there is a node exactly 1 child(say right). Suppose I was looking for sum=12 in the above diagram. This code will come at node with value 2(right child of 10) and then call hasPathSum(root->left, sum-curr). Here sum-curr = 0 (as 12=10+2). In this call root==NULL as the current node has no left child and sum asked for ==0, so it returns 1 which makes the result of hasPathSum(root->left, sum-curr)||
                                                                                                                                      hasPathSum(root->right, sum-curr) == 1 which is wrong as we cannot stop at this current node because it is not a leaf.
                                                                                                                                      So the above code can give a wrong answer when there are nodes which have only 1 child and the asked sum is reached until that node.
